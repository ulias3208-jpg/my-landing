## 1. Примеры Telegram Mini Apps

### 1.1 @wallet — Wallet (от Telegram)

**Что делает:** Официальный Mini App от Telegram. Кошелёк с P2P-маркетплейсом и Wallet Pay для оплаты товаров/услуг прямо в Telegram.

**Сильные стороны:**
- Нативное ощущение — выглядит как часть самого Telegram
- Мгновенный запуск, кэширование после первого открытия
- P2P-маркетплейс: карточки товаров, цены, кнопка «Купить»
- Встроенная оплата без перехода в сторонние сервисы
- Нативные цвета темы Telegram (светлая/тёмная)

**Слабые стороны:**
- Каталог товаров примитивный — нет категорий, фильтров, поиска
- Нет кастомизации внешнего вида для продавцов
- Нет описания услуги с мультимедиа

**Что можем сделать лучше:**
- Шаблоны витрин с фирменным стилем
- Карточки с галереей фото и видео-превью
- Система отзывов

---

### 1.2 @DurgerKingBot — Durger King Demo (от Telegram)

**Что делает:** Официальный демо-бот от Telegram, показывающий возможности Web App API. Каталог товаров с корзиной и оформлением заказа — эталонный пример «как сделать каталог в Mini App».

**Сильные стороны:**
- Чистый интерфейс: список товаров, цена, кнопка «+» для корзины
- MainButton Telegram API — корзина внизу с итоговой суммой
- Адаптируется к теме пользователя (светлая/тёмная)
- Открытый исходный код — можно взять за основу
- Haptic feedback при добавлении товара

**Слабые стороны:**
- Минимализм до аскетизма — нет изображений товаров
- Нет категорий, навигации — всё в одном списке
- Нет описания товаров, нет страницы товара
- Нет поиска и фильтрации

**Что можем сделать лучше:**
- Фото/обложки к каждой услуге
- Категории с табами наверху
- Детальная карточка при нажатии (описание, галерея, отзывы)
- Секция «Популярное» наверху

---

### 1.3 Yclients / Dikidi — Mini Apps для записи на услуги

**Что делают:** Категория Mini Apps для записи — салоны красоты, барбершопы, студии. Клиент выбирает услугу → мастера → дату → время → бронирует.

**Сильные стороны:**
- Понятный flow: Услуга → Мастер → Дата → Время → Подтверждение
- Каталог услуг с ценами и длительностью
- Календарь с доступными слотами
- Push-напоминания через Telegram
- Не нужно скачивать отдельное приложение

**Слабые стороны:**
- Дизайн «шаблонный» — белый фон, системные шрифты, без бренда
- Нет портфолио/примеров работ в карточке
- Каталог — просто список без визуала (нет фото, иконок)
- Некоторые реализации медленные (SSR вместо SPA)
- Нет предоплаты через бот

**Что можем сделать лучше:**
- Фото/видео портфолио прямо в карточке услуги
- Кастомизация цветов и бренда (как в лендинге)
- Отзывы клиентов рядом с услугой
- Онлайн-оплата через Telegram Payment API
- Анимации и micro-interactions для премиального ощущения

---

### 1.4 TON Diamonds / NFT-витрины

**Что делают:** Mini Apps в экосистеме TON — витрины цифровых товаров (NFT, стикеры, юзернеймы). Каталог → выбор → оплата в TON.

**Сильные стороны:**
- Визуально привлекательные карточки с превью (изображения, анимации)
- Плитка/сетка товаров — удобно скроллить
- Категории и фильтры по цене/типу
- TON Connect — оплата в 1-2 клика
- Нативные анимации, smooth scrolling

**Слабые стороны:**
- Долгая загрузка при большом каталоге
- Нет текстового поиска — только фильтры
- «Вылетает» на слабых устройствах из-за тяжёлых анимаций
- UX перегружен для новичка

**Что можем сделать лучше:**
- Меньше шагов до покупки
- Полнотекстовый поиск
- Лёгкая версия без тяжёлых анимаций
- Онбординг для новых пользователей

---

### 1.5 xJetSwapBot / каталоги цифровых продуктов

**Что делают:** Mini Apps для продажи цифровых товаров — чек-листы, гайды, доступ к закрытым каналам, консультации. Витрина → выбор → оплата → автоматическая доставка.

**Сильные стороны:**
- Автоматическая доставка после оплаты (ссылка, файл, доступ)
- Витрина с карточками: название, описание, цена, кнопка «Купить»
- Минимум шагов: открыл → выбрал → оплатил → получил
- Управление каталогом через бот-команды
- Статистика продаж

**Слабые стороны:**
- Визуал минималистичный — мало места для «продажи»
- Нет видео-превью или галереи
- Все витрины выглядят одинаково
- Нет отзывов / социального доказательства
- Нет upsell/cross-sell

**Что можем сделать лучше:**
- Красивые карточки с обложкой, описанием, отзывами
- Блок «С этим покупают» (рекомендации)
- Промокоды и скидки
- Страница продавца с портфолио и рейтингом
- Кастомизация витрины (цвета, логотип)

## 3. UX-паттерны и лучшие практики

### Навигация
- **Максимум 2-3 тапа** от открытия до целевого действия
- **BackButton API** — нативная кнопка «Назад» в Telegram, не делать свою
- **Табы наверху** для категорий (не боковое меню)
- **Sticky-кнопка действия** внизу (MainButton)

### Цвета и тема
- Использовать `Telegram.WebApp.themeParams` — цвета адаптируются к теме пользователя
- `bg_color`, `text_color`, `hint_color`, `button_color`, `button_text_color`
- Можно задать фиксированный `headerColor` и `backgroundColor` через API
- Для премиального ощущения: градиенты на кнопках, мягкие тени

### Типографика
- Системные шрифты (San Francisco на iOS, Roboto на Android) — быстрее загрузка
- Или Google Fonts (Inter, Space Grotesk) — как на лендинге, для единого стиля
- Размер основного текста: 15-17px (чтобы совпадал с Telegram)

### Микро-взаимодействия
- `Telegram.WebApp.HapticFeedback` — вибрация при нажатии кнопок
- Плавные переходы между экранами (slide left/right)
- Skeleton-загрузка вместо спиннеров
- Анимация при добавлении в «корзину»

### Ограничения платформы
- Максимальная высота: фиксирована (можно развернуть через `expand()`)
- Нет поддержки мультиоконности
- Файлы загружать нельзя (только через бот)
- Платежи: Telegram Payments API или внешние ссылки
- Кэш может быть агрессивным — версионирование файлов обязательно

### Telegram Web App API — ключевые фичи

```javascript
// Инициализация
const tg = window.Telegram.WebApp;
tg.ready();
tg.expand(); // Развернуть на весь экран

// Тема
const theme = tg.themeParams;
// { bg_color, text_color, hint_color, button_color, ... }

// MainButton (кнопка внизу)
tg.MainButton.setText('Заказать за 5 000 ₽');
tg.MainButton.show();
tg.MainButton.onClick(() => { /* обработка */ });

// BackButton (назад)
tg.BackButton.show();
tg.BackButton.onClick(() => { /* навигация назад */ });

// Haptic Feedback (вибрация)
tg.HapticFeedback.impactOccurred('medium');
tg.HapticFeedback.notificationOccurred('success');

// Отправка данных в бот
tg.sendData(JSON.stringify({ service: 'landing', name: 'Иван' }));

// Данные пользователя
const user = tg.initDataUnsafe.user;
// { id, first_name, last_name, username, language_code }
```

## 5. Выводы и рекомендации для проекта

### Что объединяет лучшие Mini Apps-витрины
1. **Минимум шагов** — от открытия до целевого действия 2-3 тапа
2. **Нативный look & feel** — цвета темы Telegram, haptic feedback, MainButton
3. **Карточки с ценами** — клиент сразу видит, что сколько стоит
4. **Кнопка действия внизу** — MainButton API

### Где все проигрывают (наше преимущество)
1. **Безликий дизайн** — большинство Mini Apps шаблонные и одинаковые
2. **Нет портфолио** — клиент не видит результат до покупки
3. **Нет социального доказательства** — отзывы обычно отсутствуют
4. **Нет личности** — не чувствуется, кто стоит за витриной

### Наш план: Mini App «Юлия Самарина — Услуги»

**MVP (первая версия):**
- 5 экранов: Главная → Каталог → Детали → Заявка → Подтверждение
- Аватар + имя + статус «Доступна для проектов»
- 4 категории: Сайты, Mini Apps, AI-агенты, Нейрофотосессии
- Карточки услуг с ценами, описанием, примерами работ
- Форма заявки → данные летят в TG-бот → Юлия получает сообщение
- Тёмная тема (как лендинг) + адаптация к теме Telegram
- Vercel для хостинга (уже настроен)

**Фишки для отстройки от конкурентов:**
- Единый визуальный стиль с лендингом (шрифты, цвета, градиенты)
- Галерея работ прямо в карточке услуги
- 1-2 отзыва при каждой услуге
- Haptic feedback на кнопках
- Анимации переходов между экранами
- Персональное приветствие по имени из Telegram (`user.first_name`)

**Будущие улучшения (v2):**
- Telegram Payments (оплата предоплаты)
- Промокоды
- Трекинг статуса заказа через бот
- Push-уведомления о готовности

## 6. Экспертная оценка

### 6.1 Взгляд UX-дизайнера

#### Ошибки новичков в Mini Apps

1. **Копируют веб-сайт, а не делают приложение.** Mini App — это не уменьшенный сайт. Это интерфейс на 1 задачу. Новички пытаются впихнуть лендинг целиком — hero, about, портфолио, FAQ — в Mini App. Результат: бесконечный скролл, пользователь теряется и закрывает.

2. **Своя кнопка «Назад» вместо BackButton API.** Telegram даёт нативную кнопку назад в заголовке. Когда делают свою — она выглядит чужеродно, реагирует с задержкой, и пользователь путается: нажимать вашу или системную?

3. **Слишком много полей в форме.** Клиент пришёл из Telegram — его имя и username уже известны через API. Просить заново вводить имя, телефон, email — это трение. Каждое лишнее поле = -10% конверсии.

4. **Нет состояния загрузки.** Белый экран на 2-3 секунды = пользователь думает «сломалось» и закрывает. Skeleton-экраны или хотя бы спиннер — обязательны.

5. **Игнорируют тёмную тему.** ~40% пользователей Telegram используют тёмную тему. Если Mini App всегда белый — это слепит и выглядит чужеродно.

#### Что обязательно должно быть

- **Мгновенный первый экран** (< 1 сек) — показать контент, а не загрузку
- **Одно целевое действие на экран** — не «выбери, почитай, посмотри отзывы, заполни форму» на одном экране
- **Нативная навигация** — BackButton API, MainButton API
- **Персонализация** — «Привет, {имя}!» из `tg.initDataUnsafe.user.first_name`
- **Feedback на каждое действие** — haptic вибрация, анимация, изменение цвета кнопки
- **Понятная иерархия** — заголовок > подзаголовок > цена > действие. Глаз должен «скользить» сверху вниз без усилий

#### Что бесит пользователей

- ❌ Долгая загрузка (> 3 сек — 53% уходят)
- ❌ Невозможно вернуться назад / потеря введённых данных
- ❌ Мелкий текст и кнопки (палец = 44px минимум для тач-зоны)
- ❌ Непонятно, что произошло после нажатия кнопки (отправилось? ошибка? ждать?)
- ❌ Принудительный скролл на длинной странице без якорей
- ❌ Отсутствие цен — «цена по запросу» в Mini App = мгновенный выход
- ❌ Попытка открыть внешний браузер (теряется контекст Telegram)

### 6.2 Взгляд маркетолога

#### Что заставит записаться / оставить заявку

1. **Цена прямо в карточке.** «от 5 000 ₽» — это якорь. Человек видит цену → оценивает → принимает решение. Без цены решения нет — есть только сомнения.

2. **Социальное доказательство рядом с CTA.** Не на отдельном экране «Отзывы», а прямо в карточке услуги: «⭐ Сделала лендинг за 3 дня — результат огонь! — Анна К.» Увеличивает конверсию на 15-30%.

3. **Эффект «здесь и сейчас».** Статус «Доступна для новых проектов ●» + «Ответ в течение 2 часов» — создаёт ощущение живого человека, а не формы в пустоту.

4. **Минимум шагов до заявки.** Идеал: Открыл → Выбрал услугу → Нажал «Заказать» → Написал пару слов → Готово. 4 тапа максимум. Каждый лишний экран = потеря 20-30% аудитории.

5. **Имя клиента из Telegram.** «Привет, Анна! Выбери, что тебе нужно:» — персонализация повышает вовлечённость на 20%.

#### Элементы, повышающие конверсию

| Элемент | Влияние на конверсию | Как применить |
|---------|---------------------|---------------|
| **Цена в карточке** | +25-40% vs «по запросу» | «от 5 000 ₽» на каждой карточке |
| **Отзыв рядом с услугой** | +15-30% | 1 короткий отзыв под описанием |
| **Персонализация по имени** | +15-20% | `tg.initDataUnsafe.user.first_name` |
| **Срочность/доступность** | +10-15% | «Свободна на этой неделе ●» |
| **Портфолио в карточке** | +20-35% | 1-2 скриншота работ |
| **MainButton с ценой** | +10-20% | «Заказать за 5 000 ₽» вместо «Далее» |
| **Мгновенное подтверждение** | -50% отказов | «Заявка отправлена! Юлия ответит в течение 2 часов» |
| **Малое число полей** | +30% vs длинная форма | Только «Опишите задачу» (имя берём из TG) |

#### Воронка Mini App

```
100% — Открыли Mini App (нажали кнопку в боте)
 85% — Увидели главный экран (если загрузка < 1 сек)
 60% — Выбрали категорию / услугу
 40% — Открыли детали услуги
 25% — Нажали «Заказать»
 18% — Отправили заявку
  ~5% — Конвертировались в клиента
```

**Цель:** оптимизировать каждый переход. Самые критичные потери:
- 100% → 85%: скорость загрузки
- 85% → 60%: понятность и привлекательность карточек
- 40% → 25%: убедительность детальной страницы (портфолио + отзыв + цена)
- 25% → 18%: простота формы заявки

### 6.3 Взгляд Telegram-разработчика

#### Ограничения платформы

1. **`sendData()` работает только 1 раз.** После вызова Mini App закрывается. Нельзя отправить данные и продолжить работу. Решение: для сложных сценариев использовать fetch к своему серверу вместо sendData.

2. **Размер данных sendData — max 4096 байт.** Для простой заявки (услуга + описание) хватит, но для корзины с 10+ позициями — нет. Решение: отправлять через свой API.

3. **Кэширование агрессивное.** Telegram кэширует Mini App жёстко. После обновления пользователи могут видеть старую версию часами. Решение: версионирование файлов (`app.js?v=2`) или хэши в именах.

4. **Нет доступа к файловой системе.** Пользователь не может загрузить файл (ТЗ, бриф, референс) через Mini App. Решение: после заявки бот присылает сообщение «Прикрепите файлы в ответ на это сообщение».

5. **Платежи.** Telegram Payments требует подключение платёжного провайдера (ЮKassa, Stripe). Для MVP проще без оплаты — просто заявка. Добавить позже.

6. **Viewport.** Mini App открывается не на весь экран. Нужно вызвать `tg.expand()` сразу при загрузке. Иначе — пол-экрана контента, пол-экрана пустоты.

7. **iOS vs Android различия.** На iOS MainButton может перекрывать контент. Нужен `padding-bottom: 100px` на последнем экране. Android рендерит шрифты иначе — тестировать на обоих.

#### Что работает хорошо

- ✅ **MainButton** — нативная кнопка внизу, идеально для CTA. Пользователи привыкли к ней.
- ✅ **BackButton** — нативный «назад», не нужно делать свой. Работает с жестом свайпа на iOS.
- ✅ **HapticFeedback** — вибрация при нажатии даёт ощущение нативного приложения. Используйте `impactOccurred('light')` для кнопок, `notificationOccurred('success')` для подтверждений.
- ✅ **themeParams** — автоматическая адаптация к теме пользователя. Задать `var(--tg-theme-bg-color)` в CSS.
- ✅ **initDataUnsafe.user** — имя, username, язык пользователя доступны сразу без авторизации.
- ✅ **CloudStorage** — до 1024 ключей, можно сохранять состояние (корзина, черновик заявки) между сессиями.

#### Что лучше не делать

- ❌ **Тяжёлые фреймворки** — React с полным бандлом = 3+ сек загрузка. Для 5 экранов хватит Vanilla JS или Preact (3KB).
- ❌ **Внешние шрифты без preload** — Google Fonts без оптимизации = FOUT (мигание шрифта). Либо `<link rel="preload">`, либо системные шрифты.
- ❌ **Тяжёлые изображения** — сжимать до WebP, максимум 100-200KB на изображение. Lazy loading для галереи.
- ❌ **Горизонтальный скролл** — на мобильном это баг, не фича. Карусель допустима только со свайпом.
- ❌ **window.open / внешние ссылки** — выкидывает из Telegram. Если нужна внешняя ссылка — `tg.openLink()` или `tg.openTelegramLink()`.
- ❌ **localStorage для критичных данных** — может очиститься. Для важного — `CloudStorage` API.
- ❌ **SPA-роутер с hash** — Back-кнопка Telegram не понимает hash-роутинг. Управлять состоянием вручную через стек экранов.

#### Рекомендуемая архитектура

```
mini-app/
├── index.html          — единственный HTML (как лендинг!)
├── style.css           — стили с CSS-переменными TG-темы
├── app.js              — SPA-логика (роутинг, рендер)
├── data/
│   └── services.json   — каталог услуг (легко обновлять)
└── img/
    └── portfolio/      — скриншоты работ (WebP, < 200KB)
```

### 6.5 Критический вывод

**Главное конкурентное преимущество этого Mini App — личность.**

95% Mini App-витрин в Telegram — безликие каталоги. Они выглядят как база данных с кнопкой «Купить». Ни одна не отвечает на вопрос «Кто это сделает и можно ли ему доверять?»

Mini App Юлии должен с первого экрана говорить:
- Вот я (аватар, имя, статус)
- Вот что я делаю (карточки с ценами)
- Вот результат (портфолио в карточках)
- Вот что говорят клиенты (отзыв рядом с услугой)
- Вот как начать (одна кнопка)

**Это не каталог. Это цифровая визитка с кнопкой «Купить».**
