# BRIEF: Telegram Mini App — Каталог услуг Юлии Самариной
> Пошаговый план разработки | На основе research.md (экспертная оценка UX, маркетинг, TG-разработка)

---

## 0. Концепция

**Что это:** Telegram Mini App — цифровая визитка с каталогом услуг и кнопкой «Заказать».

**Сценарий:** Клиент открывает TG-бота → нажимает кнопку «Каталог услуг» → видит услуги с ценами → выбирает → оставляет заявку → Юлия получает сообщение в Telegram.

**Принцип:** Это приложение, не сайт. Минимум текста, максимум визуала, нативный Telegram look.

|----------|-----------|

| MainButton «Отправить» | Валидация → отправка → Попап «Готово» |

### Валидация

- Textarea не пустое (минимум 10 символов)
- Если пусто — haptic `notificationOccurred('error')` + красная обводка
- Если ОК — MainButton меняется на «Отправляем...» (disabled) на время запроса

### Отправка данных

```javascript
// Данные заявки
const order = {
  service: 'Лендинг',        // выбранная услуга
  price: 'от 5 000 ₽',       // цена
  message: textarea.value,    // описание задачи
  user: {
    id: tg.initDataUnsafe.user.id,
    name: tg.initDataUnsafe.user.first_name,
    username: tg.initDataUnsafe.user.username
  }
};

// Вариант A (MVP): sendData — бот получает данные
tg.sendData(JSON.stringify(order));

// Вариант B (лучше): fetch к серверу — Mini App не закрывается
fetch('/api/order', {
  method: 'POST',
  body: JSON.stringify(order)
});
```

**Для MVP используем fetch к Telegram Bot API напрямую** (как в форме на лендинге):
```javascript
fetch(`https://api.telegram.org/bot${TOKEN}/sendMessage`, {
  method: 'POST',

    chat_id: CHAT_ID,
    text: `Новая заявка из Mini App!\n\nУслуга: ${order.service}\nЦена: ${order.price}\nКлиент: ${order.user.name} (@${order.user.username})\n\nОписание:\n${order.message}`,
    parse_mode: 'HTML'
```

### Управление навигацией

```javascript
// Стек экранов (без hash-роутера!)
const screenStack = ['main'];

function navigate(screen) {
  screenStack.push(screen);
  renderScreen(screen);
  tg.BackButton.show();
  tg.HapticFeedback.impactOccurred('light');
}

function goBack() {
  screenStack.pop();
  const prev = screenStack[screenStack.length - 1];
  renderScreen(prev);
  if (screenStack.length === 1) tg.BackButton.hide();
}

tg.BackButton.onClick(goBack);
```

## 10. Технический стек

| Компонент | Выбор | Почему |
|-----------|-------|--------|
| **Язык** | Vanilla JS | Скорость загрузки < 1 сек, без бандлера |
| **Стили** | CSS с переменными TG | Нативный look, автотема |
| **Данные** | JSON-файл | Легко редактировать, не нужен бэкенд |
| **Отправка заявок** | fetch → Telegram Bot API | Как на лендинге, проверено |
| **Хостинг** | Vercel | Уже настроен |
| **Бот** | Существующий @lending8_bot | Добавить кнопку Mini App |

### Telegram Web App API — что используем

```javascript
const tg = window.Telegram.WebApp;

// Инициализация
tg.ready();
tg.expand();                          // На весь экран
tg.headerColor = '#0A0A0F';          // Тёмный хедер

// Пользователь
const user = tg.initDataUnsafe.user;  // { first_name, username, id }

// Навигация
tg.BackButton.show() / .hide();
tg.BackButton.onClick(callback);

// CTA-кнопка
tg.MainButton.setText('Заказать от 5 000 ₽');
tg.MainButton.show() / .hide();
tg.MainButton.onClick(callback);
tg.MainButton.showProgress() / .hideProgress();

// Haptic
tg.HapticFeedback.impactOccurred('light');      // тап по кнопке
tg.HapticFeedback.notificationOccurred('success'); // заявка отправлена
tg.HapticFeedback.notificationOccurred('error');   // ошибка валидации
```

## 12. Чего НЕ будет в первой версии (MVP)

| Фича | Почему откладываем | Когда добавить |
|-------|-------------------|----------------|
| **Telegram Payments** | Нужен платёжный провайдер (ЮKassa), усложняет MVP | v2, когда будет поток заявок |
| **Промокоды и скидки** | Нет потока клиентов для тестирования | v2 |
| **Корзина (несколько услуг)** | Клиент обычно берёт 1 услугу | v2, если будет спрос |
| **Трекинг статуса заказа** | Нужна бэкенд-логика | v2 |
| **Онлайн-запись по календарю** | Сложная интеграция, Юлия ведёт вручную | v3 |
| **Админ-панель** | Данные в JSON — редактируем руками | v2, при росте каталога |
| **Поиск по услугам** | 4 категории — поиск не нужен | Никогда (мало позиций) |
| **Мультиязычность** | Аудитория русскоязычная | v3, при выходе на EN |
| **Push-уведомления** | Бот уже может слать сообщения | v2 |
| **Аналитика** | Достаточно считать заявки вручную | v2, простой счётчик |
| **Анимированные лоттие** | Тяжело для загрузки | v2, после оптимизации |
| **Видео в галерее** | Тяжело, тормозит | v3 |

## 13. Метрики успеха

| Метрика | Цель MVP | Как измерять |
|---------|----------|--------------|
| Время загрузки | < 1.5 сек | DevTools, Lighthouse |
| Конверсия «открыл → заявка» | > 15% | Счётчик заявок / открытий бота |
| Заявок в неделю | > 3 | Считать сообщения в TG |
| Bounce rate (закрыл сразу) | < 20% | Логировать `tg.ready()` vs заявки |
